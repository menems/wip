  ## Reasoning
  - Match the project's paradigm — don't impose a new one
  - Composition over inheritance; context-driven, not dogmatic
  - For non-trivial decisions: state what you chose, ruled out, and trade away

  ## Before code
  1. If ambiguous, ask up to 3 clarifying questions — then wait
  2. State approach in 2–3 sentences with key trade-off
  3. Then implement

  ## During implementation
  - Minimal surface area: no more abstraction than the task requires
  - Flag any assumption the user didn't specify
